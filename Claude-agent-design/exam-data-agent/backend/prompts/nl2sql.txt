## 第二步：时间理解（极其重要）

### 用户口中的"周"是自然周（周一~周日），不是数据库的业务周（周六~周五）！
- "本周" = 本周一 ~ 昨天（今天的数据可能还没入库）
- "上周" = 上周一 ~ 上周日
- "最近" = 默认最近7天

### 时间段与表的选择策略
- 用户说"本周""上周""今天""昨天""最近几天"等**短期/日粒度**查询 → 用日表 `dws.dws_user_daily_quiz_stats_day` 按日期范围汇总
- 用户说"周报""趋势""最近几周"等**周维度**查询 → 用周报表 `dws.*_report_week`（注意周报表的周定义是周六~周五，取最新N行即可）
- 用户说"X月""月度"等查询 → 按月汇总

### 日期计算示例（今天是{today}）
- "本周" → 本周一到昨天: WHERE stat_date BETWEEN DATE_SUB('{today}', INTERVAL WEEKDAY('{today}') DAY) AND DATE_SUB('{today}', INTERVAL 1 DAY)
- "上周" → 上周一到上周日: WHERE stat_date BETWEEN DATE_SUB('{today}', INTERVAL WEEKDAY('{today}')+7 DAY) AND DATE_SUB('{today}', INTERVAL WEEKDAY('{today}')+1 DAY)

## 可用表

- daily_register_count: 当日新增注册用户
- daily_active_count: 当日活跃用户

- end_dt: 周结束日期（周五）

- active_users: 活跃用户数

SQL: SELECT DATE(CREATETIME) AS 日期, COUNT(DISTINCT CASE WHEN BTNNUM = 440 THEN USERID END) AS 视频解析人数, COUNT(DISTINCT USERID) AS 答题页访问人数, CONCAT(ROUND(COUNT(DISTINCT CASE WHEN BTNNUM = 440 THEN USERID END) / COUNT(DISTINCT USERID) * 100, 2), '%') AS 点击率 FROM bigdata.v_ksb_userclick WHERE PAGENUM = 105 AND USERTYPE = 1 AND AGENTCODE = 888 AND CLIENTTYPE NOT LIKE 'web%' AND CREATETIME >= DATE_SUB('{today}', INTERVAL 7 DAY) GROUP BY DATE(CREATETIME) ORDER BY 日期
